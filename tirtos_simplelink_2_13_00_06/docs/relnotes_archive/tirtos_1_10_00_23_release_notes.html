Introduction

This is the General Availability (GA) release of TI-RTOS (previously called MCU SDK). This version ot TI-RTOS requires CCS 5.4.

The TI-RTOS delivers components that enable engineers to develop applications on Texas Instruments micro-controller devices. The product is comprised of multiple software components and examples of how to use these components together.

The TI-RTOS gives developers a one-stop RTOS solution for developing applications for TI embedded microcontrollers. It provides an OS kernel, communications support, drivers, and more. It is tightly integrated with TI’s Code Composer Studio development environment. In addition, examples are provided to demonstrate the use of each functional area and each supported device and as a starting point for your own projects.

Upgrade and Compatibility Information

When migrating an application that uses TI-RTOS 1.01.00.25 to TI-RTOS VERSTRDOTSUFF, the following steps must be taken:
  • New MWare: This version of TI-RTOS has a new version of MWare The following changes must be made to existing TI-RTOS applications that are based on MWare
  1. Libraries: The MWare libraries (e.g. "${COM_TI_RTSC_MCUSDK_INSTALL_DIR}/products/MWare_v140x/MWare/driverlib/ccs/Debug/driverlib.lib") must be changed to reflect the new version (e.g. "${COM_TI_RTSC_TIRTOS_INSTALL_DIR}/products/MWare_v200a/MWare/driverlib/ccs/Debug/driverlib.lib")
  2. Include Paths: The MWare include paths (e.g. "${COM_TI_RTSC_MCUSDK_INSTALL_DIR}/products/MWare_v140x/MWare") must be changed to reflect the new version (e.g. "${COM_TI_RTSC_TIRTOS_INSTALL_DIR}/products/MWare_v200a/MWare").
  • TivaWare: This version of TI-RTOS has TivaWare instead of StellarisWare. The following changes must be made to existing TI-RTOS applications that used StellarisWare
    1. Libraries: The StellarisWare libraries (e.g. "${COM_TI_RTSC_TIRTOS_INSTALL_DIR}/products/StellarisWare_9885a/driverlib/ccs-cm4f/Debug/driverlib-cm4f.lib") must be changed to point to the TivaWare libraries (e.g. "${COM_TI_RTSC_TIRTOS_INSTALL_DIR}/products/TivaWare_C_Series-1.0/driverlib/ccs/Debug/driverlib.lib")
    2. Include Paths: The StellarisWare include paths (e.g. "${COM_TI_RTSC_TIRTOS_INSTALL_DIR}/products/StellarisWare_9885a") must be changed to point to the TivaWare path (e.g. "${COM_TI_RTSC_TIRTOS_INSTALL_DIR}/products/TivaWare_C_Series-1.0").
    3. Defines: In Project Properties->Arm Compiler->Advanced Options->Prefined Symbols, the following must be changed
        • PART_LM4F120H5QR->PART_TM4C123GH6PM
        • PART_LM4F232H5QD->PART_TM4C123GH6PGE 
        • Additionally "TIVAWARE" must be defined for the TM4C boards.
    4. USB reference modules: Please refer to the new USB examples for any differences.
    5. ti.platform names: In Project Properties->General->RTSC 
      • ti.platforms.stellaris:LM4F120H5QR -> ti.platforms.stellaris:TM4C123GH6PM
      • ti.platforms.stellaris:LM4F232H5QD-> ti.platforms.stellaris:TM4C123GH6PGE
  • <Driver>_config: To allow different types of  a peripheral, each driver's config structure changed (e.g. UART_Config). The structure now has a function table, device specific objects and device specific hwAttrs. Please refer to the "Board.c" files (e.g. TMDXDOCKH52C1.c) to see an example.
  • Board.c file layout: The Board.c files (e.g. TMDXDOCKH52C1.c) layout has changed. It now uses defines to determine which peripherals are being used. For example, if UART is used in the configuration file, a TI_DRIVERS_UART_INCLUDED constant is defined. There is a #if TI_DRIVERS_UART_INCLUDED around the UART code in the board.c file.
  • I2C bitrate: This was initialized in the "Board.c" files (e.g. TMDXDOCKH52C1.c), but now is a field in the I2C_Params structure. Existing code must be re-compiled against the updated I2C.h header.
  • Watchdog_open API change: The reloadValue value was removed from the Watchdog_open() API. It is now part of the hwAttrs structure in Watchdog_config.
  • NIMUDeviceTable: The NIMUDeviceTable was in the EMAC library. It is now in the "Board.c" files (e.g. TMDXDOCKH52C1.c). 
  • CCS 5.4:  CCS 5.4 is required.
  • XDCtools location: The XDCtools product is no longer under the <TIRTOS_Install_dir>\products directory. If the exact version of XDCtools is not installed, the installer will install it in the same directory as TI-RTOS.
  • LM3 devices are no longer supported: Also, StellarisWare is no longer included in the installed product.
  • SysFlex deprecation: While still included in the product, the SysFlex module is being deprecated. Please use xdc.runtime.SysCallback instead.

Version Information

All releases have 4 digits (M.mm.pp.bb). This includes GA and pre-releases (engineering, EA, beta, etc.). Pre-releases are denoted with a suffix (e.g. 1.01.00.15_eng or 1.01.00.21-beta).

This product's version follows a version format, M.mm.pp.bb, where M is a single digit Major number, mm is 2 digit minor number, pp is a 2 digit patch number, and b is an unrestricted set of digits used as an incrementing build counter.

To support multiple side-by-side installations of the product, the product version is encoded in the top level directory, ex. tirtos_1_00_01_11.

Subsequent releases of patch upgrades will be identified by the patch number, ex. TI-RTOS 1.00.00.68 with directory tirtos_1_00_01_70. Typically, these patches only include critical bug fixes.

Please note that version numbers and compatibility keys are NOT the same. For an explanation of compatibility keys, please refer to the 'Upgrade and Compatibility Information' section.
