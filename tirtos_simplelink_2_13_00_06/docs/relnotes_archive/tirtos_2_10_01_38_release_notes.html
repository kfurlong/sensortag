Introduction

This is the GA release of TI-RTOS 2.10.01.

The TI-RTOS delivers components that enable engineers to develop applications on Texas Instruments micro-controller devices. The product is comprised of multiple software components and examples of how to use these components together. The TI-RTOS gives developers a one-stop RTOS solution for developing applications for TI embedded microcontrollers. It provides an OS kernel, communications support, drivers, and more. It is tightly integrated with TIs Code Composer Studio development environment. In addition, examples are provided to demonstrate the use of each functional area and each supported device and as a starting point for your own projects. What's New

Please refer to the Getting Started Guide and/or User Guide for more detail about the following new features.
  • New version of TI-RTOS Kernel/Networking/IPC/Instrumentation (all TI-RTOS products)
  • New PWM for TivaC devices
  • New UART (with DMA) drivers for TivaC devices
  • TSL/SSL Integration: the CyaSSL embedded SSL library is a lightweight SSL/TLS library written in ANSI C and targeted for embedded, RTOS, and resource-constrained environments - primarily because of its small size, speed, and feature set. This commercial release of the CyaSSL product is offered by Texas Instruments under a TI license for users who wish to incorporate SSL into a product running on a TI device. Upgrade and Compatibility Information

When migrating an application that uses TI-RTOS 2.00.01 to TI-RTOS 2.10.01.38, the following steps must be taken:
  • Product Version: Existing applications must point to the new corresponding "TI-RTOS for xxx" product in Project Properties->General->RTSC->Product and Repositories
  • SDSPI: SDSPITiva_HWAttrs, SDSPIUSCIA_HWAttrs, and SDSPIUSCIB_HWAttrs structures changed. Please refer to the new structure definition and change your code as needed.
  • Watchdog: The WatchdogMSP430_HWAttrs structure changed. The clockSource and clockDivider fields were changed from enums to uint8_t. The enums were removed.
  • Driver XDC Modules: The xdc.useModule of drivers in the application .cfg files are not longer required (but permitted). If they are present, a build warning occurs with instructions.
  • ENV.h: The ENV.h file was removed. The logic is now handled in the driver implementations.
  • Parameter Structures: Several of the driver's parameter structures changed. Additional fields were added to the end. If <driver>_Params_init() is used to initialize the structure and the application is recompiled against the new version of TI-RTOS, there are no breaks in compatibility.

Known Issues

There are several know issues with this release.

The following use cases are discussed in more detail (and work-arounds) on the following wiki site: http://processors.wiki.ti.com/index.php/TI-RTOS_Support
  • GCC and IAR users: By default semi-host is enabled for both GCC and IAR users. Many of the TI-RTOS examples will not run without the IDE attached (e.g. after a power-reset).
  • IAR 7.20 users: There is a problem when loading both the CC3200 or Tiva devices. Please replace the C:\Program Files (x86)\IAR Systems\Embedded Workbench 7.x\arm\bin\armlmftdi.dll file with the one on the wiki site listed above.
  • GCC users: The TI-RTOS kernel and instrumentation examples are not getting a linker file. Simply copy one from one of the TI-RTOS driver examples.

Version Information

All releases have 4 digits (M.mm.pp.bb). This includes GA and pre-releases (engineering, EA, beta, etc.). Pre-releases are denoted with a suffix (e.g. 1.01.00.15_eng or 1.01.00.21-beta).

This product's version follows a version format, M.mm.pp.bb, where M is a single digit Major number, mm is 2 digit minor number, pp is a 2 digit patch number, and b is an unrestricted set of digits used as an incrementing build counter.

To support multiple side-by-side installations of the product, the product version is encoded inEK-TM4C1294XL the top level directory, ex. tirtos_1_00_01_11.

Subsequent releases of patch upgrades will be identified by the patch number, ex. TI-RTOS 1.00.00.68 with directory tirtos_1_00_01_70. Typically, these patches only include critical bug fixes.

Please note that version numbers and compatibility keys are NOT the same. For an explanation of compatibility keys, please refer to the 'Upgrade and Compatibility Information' section.
