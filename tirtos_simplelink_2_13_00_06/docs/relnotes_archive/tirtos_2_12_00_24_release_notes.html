Introduction

This is the GA (production) release of TI-RTOS 2.12.00. TI-RTOS delivers components that enable engineers to develop applications on Texas Instruments micro-controller devices. The product is comprised of multiple software components and examples of how to use these components together. TI-RTOS gives developers a one-stop RTOS solution for developing applications for TI embedded microcontrollers. It provides an OS kernel, communications support, drivers, and more. It is tightly integrated with the Code Composer Studio development environment. In addition, examples are provided to demonstrate the use of each functional area and each supported device and as a starting point for your own projects. The TI-RTOS 2.12.00 release includes the TI-RTOS for the MSP43x product only; the upcoming TI-RTOS 2.12.01 release will include the remaining TI-RTOS products (i.e., for C2000, TivaC, and SimpleLink). What's New

Please refer to the Getting Started Guide and/or User Guide for more detail about the following new features.
  • Support for MSP432 devices, including drivers and examples.
  • TI-RTOS has moved to the new MSPWare 2.00.00 product. Because of this, peripheral drivers (USCI & EUSCI) were updated, and deprecated APIs were replaced with APIs that are compliant with the new Texas Instruments coding standards.
  • CC3200 Power Management support.
  • Addition of networking examples over a serial connection using PPP.
  • Support for SimpleLink CC3000 Wi-Fi devices was dropped -- the TI-RTOS WiFi driver and the CC3000 host driver have been removed from the product.
  • Support for some older boards -- EK_LM4F120XL and ESK-LM4F232 -- were removed from the product.

Upgrade and Compatibility Information

When migrating an application that uses TI-RTOS 2.10.xx to TI-RTOS 2.12.00.24, the following steps must be taken:
  • The API calls sl_WlanDisconnect() & sl_Stop() were removed from the WiFI_close() API. The Application must call these APIs prior to calling WiFi_close().
  • Several of the driver's parameter structures changed. Additional fields were added to the end. If <driver>_Params_init() is used to initialize the structure and the application is recompiled against the new version of TI-RTOS, there are no breaks in compatibility.
  • UARTCC3200_HWAttrs and UARTTiva_HWAttrs were changed. It now supports hardware flow control options defined by DriverLib and requires an additional buffer to receive data in the background (interrupt-driven).

Known Issues

There are several know issues with this release that will be fixed in the GA release.
  • IO parking is not yet implemented for GPIO pins. This results in higher electrical current measurements in LPDS.
  • The 3rd party vendor that produces the SSL product integrated with TI-RTOS, wolfSSL, Inc. has renamed their CyaSSL product to wolfSSL, with a corresponding change to the product's API. However, this release of TI-RTOS is integrated with the old API, so users will need to download the older CyaSSL product from here.

Version Information

All releases have 4 digits (M.mm.pp.bb). This includes GA and pre-releases (engineering, EA, beta, etc.). Pre-releases are denoted with a suffix (e.g. 1.01.00.15_eng or 1.01.00.21-beta).

This product's version follows a version format, M.mm.pp.bb, where M is a single digit Major number, mm is 2 digit minor number, pp is a 2 digit patch number, and b is an unrestricted set of digits used as an incrementing build counter.

To support multiple side-by-side installations of the product, the product version is encoded inEK-TM4C1294XL the top level directory, ex. tirtos_1_00_01_11.

Subsequent releases of patch upgrades will be identified by the patch number, ex. TI-RTOS 1.00.00.68 with directory tirtos_1_00_01_70. Typically, these patches only include critical bug fixes.

Please note that version numbers and compatibility keys are NOT the same. For an explanation of compatibility keys, please refer to the 'Upgrade and Compatibility Information' section.
